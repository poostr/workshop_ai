# ADR-0001: Базовый каркас репозитория

## Статус

Принято.

## Контекст

Первая задача `INIT-001` требует подготовить минимальный каркас проекта для дальнейшей инкрементальной реализации по `BACKLOG.md`.
Нужно обеспечить:
- предсказуемую структуру каталогов;
- базовые корневые файлы для локального старта;
- место для operational-конфигов.

## Решение

Принят следующий минимальный каркас:
- `backend/` — будущий FastAPI + Alembic + backend tests;
- `frontend/` — будущий React + TypeScript SPA;
- `ops/` — инфраструктурные конфиги (включая `ops/nginx/default.conf`);
- корневые файлы: `.env.example`, `.gitignore`, `docker-compose.yml`, `README.md`, `BACKLOG.md`.

`docker-compose.yml` на этом шаге оставлен в режиме scaffold (без финального production-ready наполнения) и будет детализирован в `INIT-004`.

## Последствия

- Плюсы: единая точка входа, низкий порог старта, понятная структура для следующих задач.
- Минусы: часть конфигурации временно placeholder и должна быть доработана в задачах этапа A.
