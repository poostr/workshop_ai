Выполнено.

## Что сделано

- Расширены i18n-словари (`ru.ts`, `en.ts`) до полного набора ключей:
  - названия стадий (`stages.IN_BOX`, …, `stages.DONE`);
  - маппинг кодов ошибок backend (`errors.ERR_INSUFFICIENT_QTY`, …);
  - подписи для всех страниц, кнопок, форм;
  - экспорт/импорт.
- Исправлен API-клиент (`shared/api/client.ts`):
  - поле ответа ошибки исправлено с `detail` на `message` (соответствие backend контракту);
  - добавлены типизированные методы для всех 7 endpoint'ов v1.
- Добавлены доменные типы фронтенда (`shared/api/types.ts`):
  - `StageCode`, `STAGES`, `STAGE_INDEX`;
  - интерфейсы для всех API-ответов и запросов.
- Обновлён `AppLayout`:
  - удалена демо-ссылка на `/types/demo`;
  - заголовок стал ссылкой на главную;
  - убрана отдельная навигация (навигация через структуру страниц).
- Добавлен полноценный `MainPage`:
  - загрузка списка типов через API;
  - поиск по имени;
  - empty state с CTA;
  - карточки типов со стадиями.
- Добавлен `CreateTypePage`:
  - форма создания типа;
  - обработка `ERR_DUPLICATE_TYPE_NAME`;
  - редирект на карточку после создания.
- Обновлён `TypeDetailsPage`:
  - загрузка деталей типа;
  - отображение counts по стадиям;
  - форма перемещения (from → to, qty с ограничением);
  - секция истории с отображением групп;
  - обработка 404.
- Обновлён роутер:
  - добавлен маршрут `/types/new`;
  - порядок маршрутов учитывает приоритет `new` над `:typeId`.
- Обновлены стили (`styles.css`):
  - CSS переменные для цветов;
  - стили для всех компонентов (карточки, формы, бейджи стадий, история).
