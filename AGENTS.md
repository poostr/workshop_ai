# AGENTS.md

Этот файл содержит глобальные правила, инварианты и Definition of Done (DoD) для AI-агентов, работающих с данным проектом. Данный файл является высшим приоритетом правды для агента.

## Источники правды (в порядке приоритета)
1. `AGENTS.md` (этот файл)
2. `TECHSPEC.md`
3. `PRD.md`
4. `ROADMAP.md` -> `BACKLOG.md`
5. `ADR/*.md`
6. `CHANGELOG.md`

## Архитектурные и бизнесовые инварианты
1. **Стадии (фиксированный порядок)**: `IN_BOX -> BUILDING -> PRIMING -> PAINTING -> DONE`.
2. **Правила перемещения**: только **вперёд** по порядку стадий; `DONE` финальная стадия (из неё нельзя перемещать).
3. **Сущности**: тип миниатюр имеет уникальное имя (unique name), счётчики по стадиям (count >= 0, никогда не могут опускаться ниже нуля).
4. **История**: хранит все перемещения, append-only структура.
5. **Импорт JSON**:
   - `all-or-nothing` транзакция.
   - Слияние (merge) по имени типа: счётчики суммируются (по стадиям), история дополняется (append). 
   - `dedupe` дублирующихся записей или событий не делаем.
6. **Локализация (i18n)**:
   - В UI нет хардкода строк (RU/EN).
   - Backend возвращает строковые константы **кодов ошибок** (например, `ERR_INSUFFICIENT_QTY`), фронтенд занимается их маппингом через i18n в читаемый текст.
7. **Инфраструктура и Деплой**:
   - Запуск осуществляется через `docker-compose`.
   - Три сервиса: `nginx` (статические файлы SPA + прокси API), `app` (FastAPI backend), `postgres` (БД с volume для сохранения данных).
8. **Безопасность**:
   - Никаких секретов в коде или репозитории (всё через `.env` и переменные окружения).
   - Принцип наименьших привилегий (контейнеры работают от имени непривилегированных пользователей, `nginx-unprivileged`).

## Definition of Done (DoD) для задач
1. Каждая реализованная задача должна следовать принципам `KISS`, `DRY` и чистого кода (Clean Code).
2. Запрещено добавлять возможности, непредусмотренные текущим скоупом задачи, если нет явного требования или обсуждения.
3. Новый/измененный код покрыт тестами или исправлен так, чтобы проходить существующие тесты:
   - Если задача связана с бекендом (`BACKLOG.md` -> `backend`), нужно убедиться, что `make backend-test` завершается успешно (`exit code 0`).
4. Линтеры и форматтеры прогоняются без ошибок (`make backend-lint` / `npm run typecheck`).
5. После завершения задачи:
   - Внести соответствующие пояснения или решения в `ADR/*.md`.
   - Добавить лог об изменениях в `CHANGELOG.md` (на русском языке).
   - Изменить маркер задачи в `BACKLOG.md` с `[ ]` на `[x]`.
   - Описать выполнение в `tasks/[TASK-ID].md` (включая ссылки на ADR).
6. Сделать полный коммит с проделанной работой.
