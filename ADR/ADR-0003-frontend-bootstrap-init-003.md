# ADR-0003: Frontend bootstrap для INIT-003

## Статус

Принято.

## Контекст

Задача `INIT-003` требует:
- поднять frontend на `React + TypeScript`;
- подготовить роутинг и базовый layout;
- заложить i18n (RU/EN) без хардкодов;
- добавить каркас API-клиента;
- обеспечить сборку в `dist` и рабочий dev-режим с заглушками `Main` и `TypeDetails`.

## Решение

1. Выбран стек bootstrap:
   - `Vite` как сборщик и dev-server;
   - `react-router-dom` для роутинга;
   - `react-i18next` + `i18next` для локализации.
2. Сформирована минимальная архитектура frontend:
   - `src/app` для композиции приложения и роутера;
   - `src/pages` для экранов;
   - `src/shared` для i18n и API-клиента.
3. Реализован скелет API-клиента с централизованной обработкой error code, чтобы в следующих задачах
   расширять его без дублирования вызовов `fetch`.
4. Для этапа bootstrap build-скрипт зафиксирован как `vite build` (без отдельного typecheck в build),
   а задача на выделенный quality gate typecheck вынесена в backlog (`INIT-006`).

## Последствия

- Плюсы:
  - frontend сразу запускается и собирается;
  - есть база для будущих UI/API задач без архитектурного долга;
  - локализация встроена в каркас и поддерживает RU/EN с первого шага.
- Минусы:
  - строгая проверка типов пока не встроена в build-конвейер и будет добавлена отдельной задачей.
