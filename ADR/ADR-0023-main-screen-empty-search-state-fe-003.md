# ADR-0023: Разделение empty state и empty search на главной странице (FE-003)

## Статус

Принято.

## Контекст

Основная функциональность главной страницы (список типов, сортировка по имени, поиск, empty state с CTA) была реализована в рамках FE-001. Однако при проверке DoD задачи FE-003 обнаружен UX-дефект: когда поиск не возвращает результатов, отображалось то же сообщение, что и при пустой базе — «Типов пока нет.» Это вводит пользователя в заблуждение, поскольку типы существуют, просто не соответствуют запросу.

## Решение

Добавлен отдельный i18n-ключ `pages.main.emptySearch` для состояния «поиск не дал результатов»:
- RU: «Ничего не найдено.»
- EN: «No results found.»

Ключ `pages.main.empty` сохранён для состояния «типов нет вообще» с CTA-кнопкой.

Альтернатива — единое сообщение с динамическим текстом — отклонена: два состояния семантически различны (пустая база vs. пустая фильтрация) и требуют разного оформления (CTA-кнопка есть только в первом случае).

## Последствия

- Пользователь получает корректную обратную связь при фильтрации.
- Два empty-состояния чётко разделены по i18n-ключам и оформлению.
