# ADR-0026: Отображение истории перемещений (FE-006)

## Статус
Принято

## Контекст
Компонент `HistorySection` был создан в рамках FE-001 как минимальный каркас.
Задача FE-006 требует корректного отображения сгруппированных записей истории, включая пограничные кейсы.

## Решения

### 1. Обратный хронологический порядок (newest first)
Бэкенд возвращает группы в порядке `created_at ASC` (от старых к новым).
На фронтенде массив переворачивается (`[...groups].reverse()`) для отображения свежих записей вверху списка.
Это стандартный UX-паттерн для activity logs / history feeds.

### 2. Локаль-зависимое форматирование дат
Вместо `new Date(ts).toLocaleString()`, которое зависит от локали браузера,
используется явная привязка к языку приложения через `i18n.language`:
- `ru` → `ru-RU` (формат `дд.мм.гггг, чч:мм`)
- `en` → `en-US` (формат `mm/dd/yyyy, hh:mm AM/PM`)

Это обеспечивает консистентность с остальной локализацией приложения.

### 3. Семантический HTML
К элементу `<time>` добавлен атрибут `dateTime` с ISO-строкой для машинной читаемости и доступности.

## Последствия
- История отображается в привычном для пользователя порядке (последние действия наверху).
- Формат дат согласован с выбранным языком интерфейса.
- Изменения минимальны и не затрагивают API-контракт или бэкенд.
